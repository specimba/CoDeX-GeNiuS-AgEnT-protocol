# Golden Runs

Purpose: keep a tiny, versioned set of proof cases for repeatable low-energy validation.

Rules:

- keep cases small and explicit
- store expected signals and invariants, not vague intentions
- do not include real user data
- expand only after the seed proves useful

Primary seed file:

- `golden/golden-run-seed.json`

Use this folder for:

- regression-proof fixtures
- refusal-vs-grounded-answer checks
- artifact-first validation baselines
